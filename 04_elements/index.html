Block Elements

Sectioning

The elements that are reflected in the web page's outline, just like the chapter titles of a book.

Before HTML5, these elements were the body and headings. This was an implicit sectioning. However, in HTML five, we have new semantic elements. These explicit sections are: section, header, footer, nav, article and aside. We can organize as we want.

  • section: very specific section defined by the user;
  • header: exactly what the name says, there can be as many as needed but only one per section;
  • footer: exactly what the name says, there can be as many as needed but only one per section;
  • nav: a special section that allows us to establish navigational elements such as menus;
  • article: sections you consider interesting independently, like blogs;
  • aside: side information, newsletter stuff, extras etc.

Grouping

The elements that are not reflected in the web page's outline, like the contents of a chapter in a book.

  • p: pragraphs, literally what it says;
  • pre: already formatted paragraphs, respects the format you give it in the code;
  • blockquote: a quoted paragraph, avoid plagiarim you doofus. indents by default. attribute cite to link the source of the quote, and att title to name it;

  • ul: unordered list, appearing by unnumbered symbols;
  • ol: ordered list, appearing by numbers/ order choice;

  • div: a neutral block. helps with categorizing things in classes and ids for later use during CSS. full freedom and almost always has a class;
  • address: an html5 new block. a special type of paragraph that are used for author data / address;
  • main: an html5 new block.it defines the part of a web page that isn't shared by other pages in the same web. not in much use.

Inline Elements

Hyperlinks

They correspond to the 'a' tag with the attribute href to indicate the url. There are several types of URL.

  • absolute: an absolute url is the kind that already exists on the web, http://, example;
  • relative to origin: all internal web page links. can give trouble if origin is changed;
  • relative to document: all internal web page links.better use, does not give trouble. ./ start reading from the document where you are. ;
  • internal: move inside the same page. for that, use # and the name of the id given to the content.
